# Bug Report: Find Transactions by Amount Returns Invalid Data

**Bug ID:** BUG_001
**Related Test Case:** TC_FINDTRANS_001
**Module:** Find Transactions
**Severity:** Medium
**Priority:** High

## Summary
Searching for a transaction by amount returns a broken result row showing "NaN-NaN-NaN" as the date and "undefined" as the transaction description, instead of the actual matching transaction.

## Steps to Reproduce
1. Log in to ParaBank
2. Click "Find Transactions"
3. Select an account with an existing transaction
4. Enter the amount of a known transaction (e.g., 500)
5. Click "Find Transactions" under "Find by Amount"

## Expected Result
The correct transaction (date, description, debit/credit amount) should be displayed.

## Actual Result
Result row shows: Date = "NaN-NaN-NaN", Transaction = "undefined", no debit/credit values.

## Environment
- Application: ParaBank (parabank.parasoft.com)
- Browser: Google Chrome
- Date Found: 2026-07-13

## Notes
Possible front-end date formatting or data-binding issue on the search results page.
